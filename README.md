# Tri-SQL Systems Database Project

Tri-SQL Systems is a relational SQLite database project designed for a simulated technology consulting and product support business. The database models consultants, customers, service tickets, product orders, suppliers, shipments, and reporting views.

This project was completed as a university group project and demonstrates relational database design, normalization, SQL scripting, sample data insertion, and analytical query development.

## Project Overview

The goal of this project was to design and implement a normalized relational database for a simulated business environment. The database supports customer management, consultant assignments, service tickets, product inventory, product orders, supplier information, shipments, and basic reporting.

## Features

- Normalized 3NF relational database schema
- Primary keys and foreign keys
- Bridge tables for many-to-many relationships
- SQL table creation scripts
- Sample data insertion scripts
- SQL queries using JOIN, GROUP BY, UNION, INTERSECT, EXCEPT, aggregate functions, and views
- Reporting views for active service tickets and product sales analytics
